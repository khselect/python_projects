```python
    # 중도절단 데이터(e=0): 생존함수(SF) 기반 우도
    # 음의 로그 우도 반환 → Nelder-Mead minimize로 β, η 추정
```

**분석 흐름:**

```
DB 로드 → 부품 ID별 그룹화
    → 수명(시간) = (제거일 - 설치일) × 24h (제거일 없으면 현재 시각)
    → 고장 여부 = event 컬럼
    → 고장 데이터 2개 이상 확보 시 MLE 수행
    → β(형상모수), η(척도모수), B10 수명 계산
```

**B10 수명:** 10%가 고장나는 시간 = `weibull_min.ppf(0.1, c=β, scale=η)`

**결과 반환 구조:**
```json

  "part_id": {
    "beta": 1.8,
    "eta": 5000.0,
    "b10_life": 720.0,
    "plot_data": {"x": [...], "y": [...]},
    "error": null
  }
}
```

## 6. 핵심 계산 로직

### 6.1 예산 예측 (`/api/cost/budget_forecast`)

Weibull CDF 차분법으로 각 부품의 "향후 H시간 내 고장 확률" 산출:

```
P(고장) = F(a + H) - F(a)
단, a = 현재 운용 시간, H = 예측 기간 (시간)
```

- 부품별 단가 통계: 최근 2년 이력 기반 μ, σ 계산 (데이터 1개면 σ = μ × 0.25)
- 총 예상 비용: `Σ (P(고장) × μ)`
- 95% 신뢰구간: `총비용 ± 1.96 × √Σ(P(고장) × σ²)`

### 6.2 시계열 시뮬레이션 (`build_timeseries`)

Weibull 분포로 수명 샘플 생성:
```python
lifetimes = rng.weibull(beta, size=n_parts) * eta_m
```

월 m에서 교체되는 부품 = `(m-1, m] 구간에 수명 종료된 부품 수`

### 6.3 정비 우선순위 점수

```
risk_score = (usage_ratio / 100) × priority_score × (1 + cost / 100,000)
```
B10 수명 70% 이상 도달 시 대상 포함.

## 10. 프론트엔드 — 종합 대시보드 UI (`/dashboard`)

관련 파일: `templates/dashboard.html`, `templates/_navigation.html`, `static/js/dashboard.js`

### 10.1 레이아웃 구조

메인 상단 네비게이션(`_navigation.html`, 데이터 관리/종합 대시보드/수명 분석/비용 분석/비용
시뮬레이터/인포그래픽)과 별개로, `/dashboard` 페이지 내부에는 6개의 세부 지표를 전환하는
**서브 탭**이 있다. 기존에는 상단 수평 탭(Bootstrap `nav-tabs`)이었으나, 좌측
**수직 필(pill) 탭**(Bootstrap `nav flex-column nav-pills`)으로 변경되었다.

```
┌──────────────┬───────────────────────────┐
│ 부품 분포 비율 │   KPI 스탯 타일 (3~4개)     │
│ 고장 부품 순위 │  ┌──────────────────────┐ │
│ 수명 분포     │  │                      │ │
│ (히트맵)      │  │    Chart.js 차트      │  │  지표 설명 카드
│ 설치 추이     │  │                      │ │  (insight-card)
│ 고장 수명 비율 │  └──────────────────────┘ │
│ 월별 고장 추이 │                           │
└──────────────┴───────────────────────────┘
```

- 좌측 탭 메뉴: `#dashboard-tabs` (`col-lg-2`), `data-bs-toggle="pill"`
- 우측 콘텐츠: `#dashboard-tabs-content` (`col-lg-10`)
- 각 탭 패널(`.tab-pane`)은 `[KPI 행] → [차트 + 설명 카드 행]` 순으로 구성

### 10.2 KPI 스탯 타일

각 탭 상단의 `<div class="row g-3 mb-3" id="{탭}-kpis"></div>` 컨테이너에
`renderKpiRow(containerId, items)` (`dashboard.js`)가 데이터를 요약한 3~4개의
카드(`.stat-tile`)를 동적으로 렌더링한다. 예: 부품 분포 → 총 부품 수 / 부품 유형 수 /
최다 사용 부품(%). 카드 상단의 색 강조 바(`accent-*` 클래스)는 아래 팔레트를 따른다.

### 10.3 색상 팔레트 (컬러블라인드 세이프)

`dashboard.js` 상단의 `PALETTE`, `INK`, `STATUS` 상수는 색약 사용자도 인접 색을 구분할
수 있도록 검증된 순서로 고정되어 있다(카테고리컬 색상은 항상 `CATEGORICAL_ORDER` 순서로만
사용하고, 임의로 순환·생성하지 않는다).

| 역할 | 값 | 용도 |
|------|-----|------|
| 카테고리컬 (blue→orange, 8색) | `#2a78d6` … `#eb6834` | 부품 분포 도넛차트 등 |
| 강조(critical) | `#d03b3b` | 최다 고장/최고 위험 항목 하이라이트 |
| 안전(good) | `#0ca30c` | 개선/감소 지표 |
| 잉크(텍스트/그리드) | `#0b0b0b`, `#52514e`, `#898781`, `#e1e0d9` | 축, 라벨, 그리드선 |

### 10.4 차트별 개선 포인트

| 탭 | 차트 유형 | 개선 내용 |
|----|----------|-----------|
| 부품 분포 비율 | 도넛 | 카테고리컬 팔레트 고정 순서 적용, 5% 이상 구간에 퍼센트 라벨(`chartjs-plugin-datalabels`) 직접 표기, 범례 우측 배치 |
| 고장 부품 순위 | 막대 | **강조(emphasis) 패턴** — 1위만 위험색(`STATUS.critical`), 나머지는 무채색 계열로 처리해 "가장 취약한 부품"에 시선 집중 |
| 수명 분포 (히트맵) | Matrix | 발생 건수에 비례한 red 시퀀셜 알파값, 셀 내 건수 직접 표기, 하단에 농도 범례(`.heatmap-legend`) 추가 |
| 설치 추이 | 라인 | 얇은 2px 라인 + 연한 영역 채우기(fill)로 추세 가독성 향상 |
| 고장 수명 비율 | 수평 막대 | 최고 비율(가장 만성적인 부품)만 강조색 처리, % 라벨 직접 표기 |
| 월별 고장 추이 | 라인 | 위험색 계열 라인, KPI 카드에 전월 대비 증감(▲/▼, %) 표기 |

### 10.5 데이터 흐름

1. `initializeDashboardTabs()`가 탭 버튼(`#dashboard-tabs .nav-link`)에 `shown.bs.tab`
   리스너를 등록하고, 최초 활성 탭의 차트를 즉시 로드한다.
2. 탭 전환 시 해당 `load*Chart()` 함수가 `/api/...` 엔드포인트를 호출한다.
3. 응답 데이터로 (a) `renderKpiRow()`로 KPI 카드를 채우고, (b) `destroyExistingChart()`로
   기존 Chart.js 인스턴스를 파괴한 뒤 새 차트를 그린다(탭 재방문 시 중복 렌더링 방지).
